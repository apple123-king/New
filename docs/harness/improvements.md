### 2026-06-10 [@improver] [测试链路]

- **触发原因**: `error-log.md` 中“测试”类问题累计达到 3 次。
- **根因判断**: 当前 Unity Skills 在 PlayMode 测试和 Play Mode 场景查询两类操作上都存在运行后句柄或批量请求不稳定的问题，导致验证证据回收比 EditMode 脆弱。
- **改进方案**:
  1. PlayMode 验证优先按“发现测试 -> 单测运行 -> 结果回查 -> 手工运行态补证”的顺序执行，不直接批量并发请求。
  2. 进入 Play Mode 后，优先单条调用 `scene_get_info` / `scene_find_objects`，避免首次就发送多条批量请求。
  3. 在 reviewer 阶段，把“测试脚本已存在但结果回收失败”与“测试本身失败”分开记录，避免误判实现质量。
  4. 如后续仍复现 `Test job not found`，优先补充替代验证链路，而不是反复重跑同一 jobId。
