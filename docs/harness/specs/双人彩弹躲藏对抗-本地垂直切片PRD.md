# 双人彩弹躲藏对抗 - 本地垂直切片 PRD

## Problem Statement

当前《双人彩弹躲藏对抗》已经有清晰的玩法幻想，但还没有验证这套核心对抗在实际操作中是否真的成立。

我们需要先回答几个高风险问题：

- 攻击方看到不到真实目标时，是否仍能通过开镜、假瞄准、打洞和换线逐步提高命中概率。
- 躲避方在看到 100% 准确枪线时，是否会形成稳定的读线、切姿势和位移博弈，而不是单纯拼反应或纯靠运气。
- 墙体可穿透且可破坏的规则，是否会带来明确的残局压力，而不是制造混乱和不可读性。
- 在不引入联网、平台接入和商业化复杂度之前，这套玩法本身是否值得继续投入到好友房实时 PvP 版本。

如果不先做本地垂直切片验证，我们很容易在核心乐趣还不稳定时，过早背上联网同步、小游戏平台适配和房间系统成本。

## Solution

先实现一个本地双人同时操作的 3D 灰盒垂直切片，用最小但完整的一套比赛流程验证玩法本身。

这个切片包含：

- 一整局最多 9 个大回合、先到 5 分获胜的完整比赛状态。
- 固定狙击位攻击方与第三人称躲避方的双视角对抗。
- 开镜即显示、对躲避方 100% 准确可见的权威枪线。
- 以镜头中心射线为权威的瞬时判定，配合可视化曳光表现。
- 砖块网格墙体、圆形范围破坏、同发穿墙继续命中。
- 同一大回合内墙体破坏状态继承，跨大回合整体重置。
- 躲避方始终可移动，可即时切换站立、下蹲、趴下三种姿势。
- 单次进攻无时间限制、攻击方可无限稳定持镜，但加入反僵局规则避免纯挂机。
- 双视角调试版 UI，用于验证玩法和节奏，不验证真实信息保密。

这份 PRD 只服务于玩法验证，不追求上线完成度。

## User Stories

1. 作为攻击方玩家，我希望在固定狙击位观察整片战场，这样我能通过找人和控线建立压迫感。
2. 作为攻击方玩家，我希望未开镜时能用较大视野搜索目标，这样我能先判断局势再进入精瞄。
3. 作为攻击方玩家，我希望开镜后视野收窄且进入精确射击状态，这样我能明确从观察切换到开火决策。
4. 作为攻击方玩家，我希望开镜和开火是两个分离输入，这样我能进行假瞄准博弈，而不是只能立刻射击。
5. 作为攻击方玩家，我希望只有开镜后才能开火，这样射击承诺更明确，躲避方也能读到威胁。
6. 作为攻击方玩家，我希望开镜后有一个很短的前摇，这样开火不是零成本瞬发，双方都有博弈窗口。
7. 作为攻击方玩家，我希望每次进攻只有 3 发子弹，这样每枪都重要，打洞和骗反应才有价值。
8. 作为攻击方玩家，我希望子弹判定与我当前开镜准线完全一致，这样命中或落空都足够可信。
9. 作为攻击方玩家，我希望墙体能被打穿并持续打洞，这样我能主动塑造残局，而不是只能等对手失误。
10. 作为攻击方玩家，我希望同一大回合内前一次进攻留下的墙洞会被保留，这样我的前几枪能为后续进攻创造价值。
11. 作为攻击方玩家，我希望即使没完全打出大洞，也能通过完整墙体直接命中躲避方，这样枪线压迫不会失真。
12. 作为攻击方玩家，我希望在长时间不开枪时系统会提醒并最终强制换手，这样整局不会因为僵持彻底停住。
13. 作为躲避方玩家，我希望在攻击方开镜时看到一条 100% 准确的连续枪线，这样我能围绕可读信息做判断。
14. 作为躲避方玩家，我希望枪线从开镜瞬间就出现并持续更新，这样我能提前读到攻击方意图。
15. 作为躲避方玩家，我希望始终可以移动，而不是被迫站桩等待射击，这样躲藏和转位才成立。
16. 作为躲避方玩家，我希望可以即时切换站立、下蹲、趴下三种姿势，这样我能通过身体高度变化躲线。
17. 作为躲避方玩家，我希望姿势切换的判定立即生效，这样不会出现“我已经蹲下却仍被旧姿势击中”的困惑。
18. 作为躲避方玩家，我希望部署阶段就能自由移动和找位置，这样我不是在菜单里选点，而是在真实空间里做决定。
19. 作为躲避方玩家，我希望部署结束时我停在哪里就从哪里开始本次进攻，这样部署行为和正式对抗是连续的。
20. 作为躲避方玩家，我希望普通移动就能在连续场地内自由穿行，这样玩法重心更集中在读枪线和躲线本身。
21. 作为躲避方玩家，我希望不直接看到攻击方剩余子弹数字，这样记枪和骗枪仍然是乐趣的一部分。
22. 作为双方玩家，我希望一局比赛包含完整的比分、大回合、先手交替和胜负结算，这样我们验证的是完整节奏而不是单个片段。
23. 作为双方玩家，我希望每个大回合只在有人命中时结束并产出 1 分，这样整局节奏围绕真正的命中结果推进。
24. 作为双方玩家，我希望墙体在进入下一个大回合时完整重置，这样每个新大回合都有清晰开局。
25. 作为测试玩家，我希望双视角调试版能同时看到攻防状态，这样我们能更快判断玩法问题来自哪里。
26. 作为测试玩家，我希望攻击方在等待对手部署时看到明确倒计时，这样我们能稳定评估 4 秒部署是否合适。
27. 作为测试玩家，我希望比赛 UI 只显示最低限关键状态，这样既方便复盘，又不会干扰玩法观察。
28. 作为设计者，我希望这版切片能明确告诉我玩法是否值得继续做联网版，这样后续投入有依据。

## Implementation Decisions

- 目标形态是本地双人同时操作的 3D 灰盒垂直切片，不实现联网、房间、平台接入。
- 比赛结构保留完整规则：最多 9 个大回合，先到 5 分获胜；若达到大回合上限仍无人先到 5 分，则按当前比分结算。
- 大回合内双方轮流进攻；只有任意一方成功命中时，大回合才结束并产出 1 分。
- 单次进攻包含 3 发子弹；若 3 发全部打空，则攻防切换并触发新的部署阶段。
- 单次进攻没有常规时间上限；攻击方允许无限稳定持镜。
- 为防止纯挂机僵持，加入反僵局规则：攻击方在可博弈状态下 8 秒未开枪时触发警告，再过 3 秒仍未开枪则本次进攻强制结束并换手。
- 攻击方采用固定狙击位，两档视野共用同一机位与连续瞄准方向。
- 未开镜时为大视野观察态，只能观察不能开火。
- 开镜后切换为窄视野瞄准态，枪线显示且允许开火。
- 开镜与开火使用两个分离输入；开镜到可开火存在 0.2 到 0.3 秒前摇。
- 命中权威由开镜镜头中心射线决定；枪口只承担火光和曳光等视觉表现。
- 开火使用瞬时射线权威判定，同时播放短曳光或飞行表现。
- 后坐力只做视觉反馈，不允许改变权威轨迹与真实命中方向的一致性。
- 躲避方视角为第三人称跟随视角，能稳定观察枪线、墙体破坏和自身位移路线。
- 躲避方全程可移动，部署阶段和被进攻阶段都不冻结。
- 躲避方姿势只有站立、下蹲、趴下三种；切换即时生效，不加入体力、前摇、姿势锁定或复杂动作惩罚。
- 命中判定首版使用单一受击体积，但该体积会随站立、下蹲、趴下状态切换形状或高度。
- 战场采用弱分区的连续躲避场地，只保留三条横向主要掩体带的空间认知，不把它们作为强规则分区。
- 每条主要掩体带放置 1 面主砖墙，三条带采用同构灰盒布局。
- 墙体采用固定砖块网格数据结构；单发子弹按命中点周围圆形半径破坏一片砖块。
- 子弹命中墙体后会先破坏砖块，再沿原轨迹继续检测躲避方；穿墙前后不偏移、不减速、不变线。
- 同一大回合内的墙体破坏状态在双方多次换手中完整继承；进入下一大回合时整体重置。
- 部署阶段固定为 4 秒。
- 部署时躲避方直接进场自由移动；倒计时结束时当前位置和当前姿势自动成为本次进攻起始状态。
- 部署结束时如果角色停在暴露通道或带间区域，该位置也视为合法起始位。
- 双视角调试版不验证真实信息保密，只验证流程遮罩和玩法状态机。
- 最低限比赛 UI 只显示：当前比分、当前大回合编号、当前谁在进攻、当前攻击方剩余子弹数、部署阶段倒计时。
- 剩余子弹数只显示给攻击方视角，不显示给躲避方视角。
- 命中结算不做短回放，只做即时彩漆命中反馈、比分更新和下一大回合切换。
- 建议拆分的核心模块包括：比赛状态机、回合与换手流程、攻击方瞄准与射击、权威轨迹显示、墙体破坏网格、躲避方移动与姿势、双视角调试 UI、玩法观察埋点与调试信息。

## Testing Decisions

- 这份切片的测试目标不是验证实现细节，而是验证对外可见行为是否稳定、可读且可复现。
- 优先把复杂逻辑放进可独立测试的纯 C# 模块，例如比赛状态机、回合切换规则、反僵局计时、墙体破坏网格、姿势对应受击体积切换规则。
- 单元测试重点覆盖：
  - 大回合和整局胜负规则。
  - 先手交替和攻防换手流程。
  - 3 发耗尽后的换手行为。
  - 8 秒警告与再 3 秒强制换手的反僵局规则。
  - 墙体网格按圆形半径破坏的确定性结果。
  - 同一发穿墙后继续命中的判定顺序。
  - 姿势切换对受击体积的即时生效规则。
- 集成测试重点覆盖：
  - 开镜后轨迹显示与真实命中方向一致。
  - 同一大回合内墙体破坏跨换手继承，跨大回合重置。
  - 部署阶段 4 秒结束后，当前位置与姿势被正确锁定为起始状态。
  - 命中后即时加分并进入下一大回合。
- 玩法验收除功能正确外，还要观察以下标准：
  - 躲避方能稳定通过读轨迹做出有效姿势或位移反应，而不是纯拼运气。
  - 攻击方能通过假瞄准、打洞和换线逐步提高命中概率，而不是只能盲猜。
  - 墙体残局会明显改变后续博弈压力，打洞有真实战略价值。
  - 对局虽可能僵持，但不会频繁出现双方都知道最优动作是继续等待的无聊局面。
- 继续进入联网版 PRD 的门槛是：
  - 双方都能在几局内自然理解规则，不需要大量口头解释。
  - 双方都能明确描述自己刚才在博弈什么，而不是觉得主要靠运气。
  - 实际试玩没有频繁滑向无事发生的死局。

## Out of Scope

- 联网同步、好友房、断线重连、中途恢复。
- 抖音小游戏与微信小游戏平台接入。
- 广告接入、商业化流程、分享进房链路。
- 角色成长、局外养成、任何影响公平性的数值系统。
- 外观皮肤、正式美术题材定稿、正式角色包装。
- 训练教学、新手引导、正式主菜单与房间流程。
- 观战、录像、命中回放、慢镜头复盘。
- 复杂角色动画状态机、分部位受击、头身四肢差异判定。
- 真实高速弹体物理、复杂后坐力散布、枪口权威命中。
- 更复杂的多层墙、非同构地图、不对称地图平衡。

## Further Notes

- 这份 PRD 是对原始玩法规格的第一次落地收敛，目标是回答“这套玩法本身值不值得继续做”。
- 原始规格里“转区冲刺”“强分区掩体区”等概念，在本地垂直切片中已经被弱化或移除，以便把验证焦点集中到读枪线、破墙、持续移动和命中博弈。
- 如果本地切片验证结果积极，下一份 PRD 应优先讨论真实非对称信息保密、联网权威归属、房间系统与小游戏平台适配。
